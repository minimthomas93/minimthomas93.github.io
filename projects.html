🔐 Tamarin Model of IKEv2 Protocol

The Internet Key Exchange Protocol Version 2 (IKEv2) is used by IPsec to secure traffic on the network layer (see Figure 1). This project involved formally modeling the IKEv2 key exchange mechanism using the Tamarin Prover.

📘 Notation

  • INIT and AUTH are identifier strings.
  • SAi and infoi represent a set of algorithms and meta information, respectively.
  • cookie is a random value to identify the session later on.
  • xA and xB are the Diffie-Hellman shares of the two parties. g is a generator.
  • ni is a nonce.
  • k is the shared key computed from the Diffie-Hellman operation gxAyB.
  • kj is a key computed as: kdf(k, nA, nB, j).
  • transcripti is computed by hashing SAi, gxi, ni, infoi, nj, MACi, where:
    • MACi = MAC(pki, k3) for A
    • MACi = MAC(pki, k4) for B
  • For transcriptA, the cookie is added to the first position of the transcript.
  • 🔑 Modeled initialization of key pairs and identities with role-switching (Initiator/Responder).
  • 🔄 Implemented the full IKEv2 protocol with restrictions to prevent self key exchange.
  • 🔐 Enforced single session per key pair.
  • ✅ Verified successful execution through sanity checks.
  • 📜 Formulated six agreement lemmas, including message and transcript agreements.
  • 🔒 Proved key secrecy for all exchanged keys.
  • 📊 Ran simulations with multiple sessions, analyzed lemma proofs, and documented discovered attacks (if any).
