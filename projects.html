📘 Master's Thesis (In Progress)

Title: Coreografa : Feedback-directed input generation and analysis Framework

Software performance evaluation is a crucial part of modern system testing, particularly in domains where efficiency, scalability, or memory usage directly affects reliability. Traditional performance testing approaches rely on manually crafted inputs and static benchmarking, making it difficult to explore how varying input characteristics system behavior. To address this challenge, we propose Coreografa- a feedback directed framework that automates performance analysis through grammar based input generation and targeted performance exploration.

  • Key Contributions: Developed a cyclic system that:
    • Generates test inputs based on grammar, user-defined constraints
    • Measures metric (runtime/memory) performance and properties of inputs (length, sortedness etc.)
    • Analyzes results and generate new JSON requests with new metric and property values
    • Create new grammar files with new metric and property values from JSON requests
  • Use Cases: Sorting algorithms,,Searching algorithms, HTML-to-PDF conversion
  • Goal: Automatically generate inputs that reveal performance bottlenecks or efficiency improvements

This thesis is currently ongoing as part of my Master's in Cybersecurity at CISPA Helmholtz Center for Information Security.

Technologies: Python · Fandango · Fuzzing · Plotting graphs · AST parsing · WeasyPrint · JSON · GitHub

🔐 Tamarin Model of IKEv2 Protocol

The Internet Key Exchange Protocol Version 2 (IKEv2) is used by IPsec to secure traffic on the network layer (see Figure 1). This project involved formally modeling the IKEv2 key exchange mechanism using the Tamarin Prover. This project was developed as part of the Formal analysis of real-world security protocols course at Saarland University.

📘 Notation

  • INIT and AUTH are identifier strings.
  • SAi and infoi represent a set of algorithms and meta information, respectively.
  • cookie is a random value to identify the session later on.
  • xA and xB are the Diffie-Hellman shares of the two parties. g is a generator.
  • ni is a nonce.
  • k is the shared key computed from the Diffie-Hellman operation gxAyB.
  • kj is a key computed as: kdf(k, nA, nB, j).
  • transcripti is computed by hashing SAi, gxi, ni, infoi, nj, MACi, where:
    • MACi = MAC(pki, k3) for A
    • MACi = MAC(pki, k4) for B
  • For transcriptA, the cookie is added to the first position of the transcript.
  • 🔑 Modeled initialization of key pairs and identities with role-switching (Initiator/Responder).
  • 🔄 Implemented the full IKEv2 protocol with restrictions to prevent self key exchange.
  • 🔐 Enforced single session per key pair.
  • ✅ Verified successful execution through sanity checks.
  • 📜 Formulated six agreement lemmas, including message and transcript agreements.
  • 🔒 Proved key secrecy for all exchanged keys.
  • 📊 Ran simulations with multiple sessions, analyzed lemma proofs, and documented discovered attacks (if any).

Technologies: Tamarin Prover · Security Protocol Modeling · Formal Verification · Cryptographic Protocols

🧪 Automated Grammar-Based Constraint Learning

This project, developed as part of the Security Testing course, focuses on building a system that learns input constraints from positive and negative sample sets using grammar analysis. The system helps uncover abstract constraints that all valid inputs satisfy and all invalid ones violate, enabling advanced applications in fuzzing and bug detection.

  • Implemented a constraint learning engine using derivation trees and grammar rules.
  • Automatically instantiated unary/binary constraints and verified them across test inputs.
  • Generated new inputs that satisfy or violate learned constraints.

Technologies: Python · Formal Grammars · AST Parsing · Blackbox Oracle Testing
